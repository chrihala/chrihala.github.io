Fuzzing finds a lot of bugs, especially in code that parses stuff like files or urls so I think it should be an important part of my toolkit.

One of the tools I've heard a lot about is American Fuzzy Lop or AFL. (http://lcamtuf.coredump.cx/afl/) So I thought a good starting point would be to learn how to use this fuzzing tool and perhaps move from there.

When looking to learn more about fuzzing tools in the past I've looked to tools or frameworks such as SPIKE, Sulley and Peach.



Getting started 

http://lcamtuf.coredump.cx/afl/QuickStartGuide.txt

Instrumenting

If we have the sourcecode and we want to fuzz a binary we can compile it using afl-gcc or g++. This instruments the program, which basically means we make it ready for fuzzing.

Starting and Resuming a fuzz-session

The afl-fuzz program is the fuzzer itself. It can be used like this:

./afl-fuzz -i test_cases/ -o test_results/ vulnerable_binary @@

To resume an interrupted session:
./afl-fuzz -i- -o existing_output_dir [...etc...]
