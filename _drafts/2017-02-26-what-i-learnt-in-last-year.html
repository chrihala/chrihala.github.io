--- layout: post title: What I learnt in the last year date: '2017-02-26T08:48:00.002-08:00' author: Hland tags: modified_time: '2017-02-26T08:48:52.442-08:00' blogger_id: tag:blogger.com,1999:blog-3264165303439231899.post-2584951875073449830 blogger_orig_url: https://blankhat.blogspot.com/2017/02/what-i-learnt-in-last-year.html --- In the past 10 months I have been working as a Application Penetration Tester with a focus on testing, auditing and helping fix web applications. I have audited a wide range of applications made using an array of different technologies. In order to be able to read the code of these applications I had to pick up new languages, get well acquainted with the program idioms of each technology and know the technology stack from top to bottom. This is tough work.

My experience with code auditing from the past three years as a pentester doing network and application security was limited. Most of the security testing I did was blackbox and only on occasional engagements we had source code to review. 

There's no doubt that having the source code enables a security tester to find a lot more bugs. But, it also requires the tester to be well versed in a lot of different technologies. 


List of languages that applications I have audited in the past year have been written in:
  • Perl 
  • Java
  • Ruby
  • Python
  • Javascript
  • Go
  • Bash

Of these 7 languages I knew Java, Python, Javascript and Bash already. However, web applications can be written in a range of different ways. A Perl CGI script is different from a full-fledged MVC Perl application. A NodeJS application is very different from being able to read Javascript executed in the browser on a random website. 

The stack that accompanies these languages encourages (or enforces) some structure to the code. Or (as in Rails) does lots of magic behind the scenes that is just implicitly known to those familiar with the framework. So once you have audited a NodeJS application, auditing the next will most likely be easier.

Books

The way I like to learn a new language is to start reading some code and accompany it with reading the quickstart guides or tutorials. But to get a more wholistic overview of the language it is always good to pick up a book about it. Here are some of the books I've been reading while learning these languages:

  • For Perl I've been reading "Learning Perl" 
  • For NodeJS I've been reading "Professional Node.JS"
  • Go Go I've been reading "The Little Go Book"
  • For Ruby on Rails I've been reading "The Humble Little Ruby Book" and the "Rails Guides" 
These are nice supplements to the documentation pages for the language or framework.
